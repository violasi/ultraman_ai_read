# 家长操作说明书

本应用支持两个主题版本：**奥特曼日记**（默认）和
**艾莎日记**。两个版本共用同一套代码，通过环境变量切换，数据完全隔离。

## 二、启动密码保护

构建时可以设置启动密码，防止应用被随意传播：

``` bash
# 不加密码的构建（开发测试用）
npm run build:elsa

# 加密码的构建（分发用）
VITE_APP_PASSWORD=你的密码 npm run build:elsa
```

**工作原理：** - 设置密码后，打开应用时必须输入密码才能使用 -
首次输入正确密码后，会绑定当前设备（基于浏览器指纹） -
同一设备后续打开不再需要输密码 - 如果有人把 dist
文件夹拷给别人，对方打不开（不知道密码） -
清除浏览器数据后需要重新输密码

## 三、如何在妹妹的电脑上部署艾莎版

### 方法 A：拷贝构建产物（最简单，推荐）

1.  在你的电脑上运行构建命令：

    ``` bash
    VITE_APP_PASSWORD=你的密码 npm run build:elsa
    ```

2.  构建完成后，`dist` 文件夹就是完整的艾莎日记应用

3.  把整个 `dist` 文件夹拷贝到妹妹的电脑（U盘、AirDrop 等）

4.  在妹妹的电脑上，用浏览器直接打开 `dist/index.html` 即可使用

> 注意：AI 自动生成故事功能需要联网（需要调用 OpenAI
> API）。离线时可以使用"复制 Prompt 手动请 AI 写故事"模式。

### 方法 B：妹妹电脑上安装开发环境

1.  安装 [Node.js](https://nodejs.org/)（选择 LTS 版本）

2.  把项目代码复制到妹妹电脑

3.  打开终端，进入项目目录，运行：

    ``` bash
    npm install
    npm run dev:elsa
    ```

4.  浏览器打开 `http://localhost:5173` 即可使用

### 方法 C：部署到云服务

1.  运行 `npm run build:elsa`
2.  把 `dist` 文件夹部署到任意静态网站托管服务（如腾讯云
    EdgeOne、Vercel、Netlify 等）
3.  妹妹通过网址访问

## 四、日常使用

### 设置 OpenAI API Key

进入"设置"页面（右上角齿轮图标），输入 OpenAI API Key
并保存。设置后可以"一键生成"故事。不设置也能用——选择"复制 Prompt 手动请
AI 写故事"模式。

### 数据备份与恢复

-   **导出**：设置页面 → 点击"导出存档"，会下载一个 JSON 文件
-   **导入**：设置页面 → 点击"导入存档"，选择之前导出的 JSON 文件
-   建议定期导出备份，防止数据丢失

> 重要：奥特曼版和艾莎版的存档数据互不兼容，不要交叉导入。

### 如何添加绘本

把绘本数据放入 `public/books/` 目录下。每本书一个文件夹，包含
`book.json` 和 `meta.json`。绘本数据两个版本共用。

| 项目        | 奥特曼日记                 | 艾莎日记               |
|-------------|----------------------------|------------------------|
| 启动命令    | `npm run dev`              | `npm run dev:elsa`     |
| 构建命令    | `npm run build`            | `npm run build:elsa`   |
| 主题色      | 红色                       | 冰蓝色                 |
| 角色        | 9 个奥特曼                 | 9 个冰雪奇缘角色       |
| 特色页面    | 英雄馆                     | 艾莎馆                 |
| AI 故事风格 | 光之国/M78 星云            | 阿伦黛尔/冰雪王国      |
| 数据存储    | `ultraman_diary_*`         | `elsa_diary_*`         |
| 备份文件名  | `ultraman-diary-日期.json` | `elsa-diary-日期.json` |
