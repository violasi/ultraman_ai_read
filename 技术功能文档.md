# 奥特曼日记 · 技术功能文档

## 项目概览

| 项目 | 说明 |
|------|------|
| 技术栈 | React 18 + TypeScript + Vite + Tailwind CSS |
| 路由 | React Router (Hash 模式) |

## 一、三个阅读模块

### 1. 汉字阅读模块

每篇故事由逐句展示的汉字组成，每个字包含 `char`（汉字）和 `pinyin`（拼音）。

**分级规则：**

| Level | 句子数 | 每句字数 | 生字控制 |
|-------|--------|----------|----------|
| 1 | 6 句 | 5-10 字 | 每句 1-3 个生字 |
| 2 | 8 句 | 8-15 字 | 每句 1-3 个生字 |
| 3 | 10 句 | 10-20 字 | 每句 1-3 个生字 |

**显示逻辑：**
- 已知字：直接显示汉字，不标拼音
- 生字：汉字上方用 ruby 标注拼音
- 点击任意字：播放发音 + 显示拼音
- 标点符号不可点击

**智能识字判断（读完故事后触发）：**
- 生字在整篇中**没有被点击** → 自动加入"已知汉字"（判定为已学会）
- 已知字在整篇中**被点击过** → 从已知汉字移出，加入"生词本"（判定为不太熟）
- 读完后会弹出摘要，显示本次新学会和需要复习的字

### 2. 拼音阅读模块

全文以拼音形式展示，声母韵母用不同颜色区分。

**分级规则：**

| Level | 句子数 | 拼音难度 |
|-------|--------|----------|
| 1 | 5 句 | 简单声母韵母组合，少用整体认读音节 |
| 2 | 6-7 句 | 复韵母（ai, ei, iu）、前后鼻音（an, en, ing） |
| 3 | 7-9 句 | 所有拼音组合，含复杂韵母 |

**声调颜色：**
- 一声（阴平）：红色
- 二声（阳平）：绿色
- 三声（上声）：紫色
- 四声（去声）：蓝色
- 轻声：灰色

**数据结构：** 每个词包含 `pinyin`、`char`（可选汉字）、`tone`（0-4）、`meaning`（中文释义）。

### 3. 英语阅读模块

基于自然拼读（Phonics）的英语分级阅读。

**分级规则：**

| Level | 句子数 | 自然拼读规则 | 举例 |
|-------|--------|-------------|------|
| 1 | 5 句 | CVC 词 + 简单视觉词 | cat, big, run, the, is, a |
| 2 | 6-7 句 | CVCE 词 + 辅音连缀 + 常见二合字母 | make, like, stop, from, sh, ch, th |
| 3 | 7-9 句 | 复杂词、r 控制元音、元音组合 | rain, beat, book, bird, care |

**数据结构：** 每个词包含 `word`（英文）、`phonics`（音素拆分，如 `sh-i-p`）、`chinese`（中文翻译）。每句还有 `chineseTranslation`（整句中文翻译）。

## 二、测验系统

每篇故事读完后进入测验环节：

| 项目 | 规则 |
|------|------|
| 题目数量 | 3 道选择题（4 选 1） |
| 评分 | 每答对 1 题得 1 颗星 |
| 通过门槛 | 2 颗星及以上 |
| 通过奖励 | 解锁 1 张奥特曼卡片 |
| 不通过 | 无卡片，可重新阅读 |

星星会累计显示在右上角 Header 中。

## 三、奥特曼卡片系统

### 卡片数据

| 统计项 | 数量 |
|--------|------|
| 总卡片数 | 110 张 |
| 奥特曼角色数 | 54 个 |
| 普通卡 (common) | ~55 张 |
| 稀有卡 (rare) | ~39 张 |
| 传说卡 (legendary) | ~16 张 |

### 形态系统

每个奥特曼有 1-6 张卡片，对应不同形态：

| 角色 | 卡片数 | 形态举例 |
|------|--------|----------|
| 泽塔 | 6 | 原始、阿尔法装甲、贝塔冲击、伽马未来、德尔塔天爪、西格玛微光 |
| 赛罗 | 6 | 基础、强壮日冕、月神奇迹、究极赛罗、闪耀赛罗、超越赛罗 |
| 捷德 | 5 | 原始、刚燃、机敏、尊皇、究极终极 |
| 欧布 | 5 | 原生、重光、爆炎、疾风、三位一体 |
| 迪迦 | 4 | 复合型、强力型、空中型、闪耀迪迦 |
| 特利迦 | 4 | 复合型、强力型、天空型、真理形态 |
| 德凯 | 4 | 闪亮型、强壮型、奇迹型、动态型 |
| 戴拿 | 4 | 闪亮型、强壮型、奇迹型、闪耀形态 |
| 其他 | 1-3 | 基础形态 + 可能的强化形态 |

### 卡片解锁方式

| 方式 | 说明 |
|------|------|
| 故事解锁 (`story`) | 读完故事 + 通过测验（2星+）→ 随机获得一张未拥有的故事卡 |
| 成就解锁 (`milestone`) | 达成特定里程碑条件 → 解锁对应成就卡 |

### 卡片视觉

每张卡片有独立的渐变配色（`colors: [startColor, endColor]`），按稀有度有不同边框效果：
- 传说卡：金色光环 + ring
- 稀有卡：蓝色细边 + ring
- 普通卡：无额外效果

集齐同一角色全部形态后，该角色区域显示"已集齐"标志。

## 五、连续阅读追踪

| 字段 | 说明 |
|------|------|
| `currentStreak` | 当前连续天数 |
| `lastReadDate` | 最后阅读日期 (YYYY-MM-DD) |
| `longestStreak` | 历史最长连续天数 |
| `activeDates` | 最近 30 天的阅读日期列表 |

**计算规则：**
- 今天已读：连续天数不变
- 昨天有读、今天第一次读：连续天数 +1
- 超过 1 天未读：连续天数重置为 1
- 首页显示"连续阅读 N 天！"横幅

## 六、已知汉字管理

**初始预设：100 个常用汉字**

分类包括：数字(10)、方位(11)、自然(24)、人物家庭(18)、身体(7)、动物(6)、动作(9)、形容词(11)、常用字(11)、学习(4)、其他(6)

**管理方式：**
- 已知汉字页面：手动添加/删除，支持批量粘贴
- 阅读中自动调整：上述智能识字判断逻辑
- 测验自动晋升：同一个字在测验中连续答对 3 次 → 自动晋升为已知字

## 八、语音朗读 (TTS)

| 环境 | 实现 | 说明 |
|------|------|------|
| 桌面浏览器 | Web Speech API | 原生支持，无需额外配置 |

- 英文：0.75 倍速

## 九、故事工坊

### 三个标签页

| 标签 | 功能 |
|------|------|
| 生成 Prompt | 根据模块/难度/主角生成 AI 提示词，一键复制 |
| 导入故事 | 粘贴 AI 返回的 JSON，校验后导入 |
| 设置 | 填入 OpenAI API Key，查看数据统计 |

### 主角选择

下拉菜单列出全部 54 个奥特曼角色，默认"任意奥特曼（随机）"。选择具体角色后，Prompt 会指定围绕该角色创作。

### OpenAI 直接生成

| 项目 | 说明 |
|------|------|
| 模型优先级 | gpt-4o-mini → gpt-4o → gpt-3.5-turbo |
| 重试逻辑 | 404（模型不存在）时自动尝试下一个模型 |
| Temperature | 0.8 |
| 错误处理 | 401 提示 key 无效，429 提示频率限制 |

### 自定义故事校验

- 必须包含 `sentences`（非空数组）
- 必须包含 `quiz`（非空数组）
- 自动生成 ID：`custom-{module}-{level}-{timestamp}`
- 自动分配奖励卡片：优先选择未解锁的故事类卡片

## 十、游戏存档系统

### 导出（保存存档）

1. 读取全部 12 个 localStorage key
2. 生成包含 `version`、`exportedAt`、`summary`、`data` 的 JSON
3. 文件名包含进度信息：`存档_8张卡片_18星_2026-03-27.json`

### 导入（读取存档）

1. 选择 JSON 文件
2. 解析并展示存档预览（卡片数、星数、汉字数）
3. 与当前进度对比显示
4. 确认后写入 localStorage，页面刷新

### 存储的全部 Key

| Key | 内容 |
|-----|------|
| `orange_read_progress` | 故事进度、测验分数、总星数 |
| `orange_read_vocab` | 生词本 |
| `orange_read_cards` | 卡片解锁状态 |
| `orange_read_settings` | 用户设置 |
| `orange_read_known_chars` | 已知汉字列表 |
| `orange_read_custom_stories` | 自定义故事（中/拼/英） |
| `orange_read_milestones` | 成就完成状态 |
| `orange_read_streak` | 连续阅读数据 |
| `orange_read_char_encounters` | 汉字测验遇到次数 |
| `orange_read_initial_chars_count` | 初始汉字数基准线 |
| `orange_read_openai_key` | OpenAI API Key |
| `orange_read_last_workshop` | 上次使用故事工坊的日期 |

## 十一、首页功能

### 每日故事推荐

- 基于日期 + 小时的确定性随机种子
- 从 10 条预设消息中选取（包含角色 emoji + 名字 + 剧情片段）
- 同一天同一小时内显示相同内容

### 阅读入口

- 三个主模块大按钮（汉字/拼音/英文）
- 按模块浏览全部故事
- 快捷入口：生词本、已知汉字、卡片收集、故事工坊、存档

### 故事工坊提醒

当满足以下条件时显示提醒横幅：
- 距离上次使用工坊超过 7 天，或
- 总故事数少于 12 篇
